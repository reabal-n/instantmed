1. Company Overview

InstantMed Pty Ltd (ABN 64 694 559 334) is an Australian telehealth platform operating at instantmed.com.au. The platform enables patients across Australia to request medical certificates, repeat prescriptions, and GP consultations online. Registered practitioners review requests asynchronously and deliver documents digitally, with no in-person visit required.

InstantMed is a single-brand operator. It does not provide marketing or advertising services to third parties and manages only its own Google Ads account.

2. Tool Overview

InstantMed has built a lightweight, internal-only analytics integration that connects the Google Ads API to the company's development environment via the Model Context Protocol (MCP). This tool is used exclusively by the founding team (founder and CTO) to monitor ad spend efficiency and campaign performance.

What the tool does:

  • Fetches campaign performance data (impressions, clicks, cost, conversions) via GAQL queries
  • Surfaces keyword analytics and quality score metrics for optimisation review
  • Reports on cost-per-acquisition and conversion rates across service lines
  • Provides account-level spend summaries for budget planning

What the tool does not do:

  • It does not create, pause, modify, or delete any campaigns or ad groups
  • It does not adjust bids or budgets programmatically
  • It does not manage Google Ads accounts on behalf of any third party
  • It does not expose data outside the internal development environment

All campaign changes are made manually through the Google Ads UI by the internal team after reviewing analytics output from this tool.

All components run locally. No data is written to external storage. The MCP server process terminates when the development session ends.

Authentication: The MCP server authenticates using Google Application Default Credentials (ADC) via OAuth2. Credentials are stored in the standard ADC location on the developer's local machine (~/.config/gcloud/) and are never transmitted to third parties or stored in the application codebase.

Query flow:

  • The developer issues a natural-language request (e.g., "show me campaign performance this week")
  • The MCP server translates this into a typed GAQL query
  • The query is sent to the Google Ads API using read-only service methods
  • Results are returned as structured JSON and displayed in the developer's terminal/IDE
  • No data is persisted after the session ends

5. Data Handling

Data returned from the Google Ads API is used only for immediate analytical display within the developer's environment. The following principles govern all data handling:

  • No permanent storage: Query results are held in memory for the duration of the session only. No results are written to files, databases, or external services.
  • Read-only access: The integration is scoped to read-only API methods. No campaign data is modified by this tool at any time.
  • No PII beyond Google Ads audience data: The tool retrieves aggregate performance metrics (impressions, clicks, cost, conversions). It does not retrieve personally identifiable information about individual users or patients.
  • Internal use only: Results are displayed within the developer's local development environment (Claude Code / terminal). Data is not exported, shared, or published.
  • Single-account scope: All queries are scoped to InstantMed's own Google Ads account (920-501-0513). No third-party account data is accessed.

6. Security

  • OAuth2 via Application Default Credentials: Authentication follows Google's recommended ADC pattern. Credentials are stored locally in the standard ADC path and are refreshed automatically by the Google Auth Library.
  • No credential storage in source code: API keys and OAuth tokens are never committed to the codebase or stored in environment variable files checked into version control.
  • Local execution only: The MCP server runs as a local process on the developer's machine. It does not expose any HTTP endpoints and is not reachable over the network.
  • No third-party access: No contractors, vendors, or external parties have access to the credentials or the tool output. The integration is operated solely by the InstantMed founding team.
  • Minimal OAuth scopes: The OAuth consent is requested with the https://www.googleapis.com/auth/adwords scope, which is required for Google Ads API access. No additional Google scopes are requested.
  • Audit trail: All API requests pass through Google's standard access logging. No additional logging infrastructure is required given the internal, read-only nature of the tool.
